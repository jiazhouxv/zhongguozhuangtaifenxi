# 06｜民众情绪监控框架

> 民众情绪数据是中国状态分析的“症状雷达”，不是事实本身。它的价值在于更早发现官方数据和商业数据尚未完全反映的压力点，但必须与硬数据交叉验证。

---

## 1. 定位

民众情绪监控用于观察：

1. 居民对房价、房贷、就业、收入、消费、负债的心理变化。
2. 不同行业、城市、年龄层的压力扩散。
3. 官方数据滞后时的早期异常信号。
4. 政策出台后民间反馈是否改善。
5. 某些关键词是否跨平台、跨城市、跨行业同步升温。

它不能直接证明：

- 房价真实跌幅。
- 真实失业率。
- 企业真实倒闭数量。
- 银行真实坏账。
- 地方财政真实缺口。

这些必须通过硬数据验证。

## 4. 指标设计

### 4.1 基础指标

| 指标 | 说明 |
|---|---|
| keyword_frequency | 关键词出现频率 |
| keyword_growth_rate | 关键词环比/同比增长 |
| platform_breadth | 出现该主题的平台数量 |
| city_breadth | 出现该主题的城市数量 |
| industry_breadth | 出现该主题的行业数量 |
| negative_sentiment_ratio | 负面情绪占比 |
| first_person_ratio | 第一人称经历占比，越高越接近真实个体经验 |
| repost_similarity | 重复搬运相似度，越高噪音越大 |
| verified_event_count | 可被硬数据验证的事件数量 |

### 4.2 复合指标

```text
Public Sentiment Stress Score
= 关键词热度
+ 负面情绪占比
+ 跨平台扩散
+ 跨城市扩散
+ 第一人称经历占比
- 重复搬运惩罚
- 营销号/机器人噪音惩罚
```

## 5. 交叉验证规则

### 5.1 房地产情绪验证

只有当以下三类以上同时出现，才进入结论：

1. “卖不掉、降价、断供、法拍”热词升温。
2. 二手房挂牌量上升。
3. 成交周期拉长。
4. 法拍数量或流拍率上升。
5. 居民中长期贷款走弱。
6. 房产中介成交反馈恶化。

### 5.2 就业情绪验证

只有当以下三类以上同时出现，才进入结论：

1. “裁员、降薪、找不到工作、35岁危机”热词升温。
2. 招聘岗位数下降。
3. 薪资中位数下降。
4. 企业财报出现降本、裁员、利润下降。
5. 灵活就业收入下降或从业者增加。
6. 社保缴纳人数或就业质量指标走弱。

### 5.3 消费情绪验证

只有当以下三类以上同时出现，才进入结论：

1. “消费降级、平替、不敢花、存钱”热词升温。
2. 团购和低价渠道增长。
3. 快消均价下降或销量涨但销售额不涨。
4. 高端消费和奢侈品承压。
5. 商铺转让和关店上升。
6. 居民贷款下降、存款上升。

## 6. 数据质量风险

民众情绪数据存在以下风险：

1. 平台样本偏差。
2. 审查和删帖偏差。
3. 营销号、机器人和重复搬运。
4. 情绪极端者更愿意发声。
5. 城市、年龄、收入层级分布不均。
6. 单个平台热点无法代表全国。
7. 关键词可能被替代表达规避。

因此当前可信度初始设为：

```text
0.48
```

低于外资、债务、企业财务等硬数据。

---

## 7. 合规原则

1. 优先使用公开趋势、公开报告、公开网页和正式开放接口。
2. 遵守平台服务条款、版权要求和数据使用限制。
3. 不采集个人隐私。
4. 不建立个人画像。
5. 只做聚合统计，不暴露个人内容。
6. 保留数据来源、采集时间、关键词和处理规则。
7. 对每个平台单独做合规性评估。

---

## 8. 在 Dashboard 中的作用

民众情绪监控当前接入以下模块：

- 房地产真实流动性。
- 地方财政与城投债。
- 居民资产负债表。
- 就业质量。
- 消费降级。
- 企业利润与现金流。
- 产能过剩与价格战。
- 外资与资本信心。
- 社会压力外溢。
- 政策反应强度。

它不应该单独决定总分，但在“民生压力模式”和“资产负债表衰退模式”中应提高权重。
